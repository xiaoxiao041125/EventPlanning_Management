# 活动策划与管理系统 - 项目介绍

## 1. 项目概述

### 1.1 项目名称
**活动策划与管理系统**（Event Planning Management System）

### 1.2 项目定位
本系统是一个面向文化传媒公司的综合性活动管理平台，旨在通过信息化手段提升活动策划、执行、管理的效率，实现业务流程的数字化和自动化。

### 1.3 项目目标
- 实现活动申请到执行的全流程管理
- 优化人员、物料资源的调配效率
- 提供准确的财务统计和薪资管理
- 建立完善的权限管理体系
- 提升公司整体运营效率

## 3. 核心功能模块

### 3.1 用户管理模块
- **用户注册与登录**：支持普通用户和员工两种角色注册
- **权限管理**：基于角色的访问控制（RBAC）
- **用户信息管理**：个人信息查看、修改、密码重置

### 3.2 活动管理模块
- **活动申请**：用户提交活动需求
- **审批流程**：多级审批机制（业务管理员→仓库管理员）
- **进度跟踪**：实时查看活动状态
- **人员安排**：为活动分配合适的工作人员

### 3.3 物料管理模块
- **物料信息**：物料的增删改查
- **库存管理**：实时库存监控
- **出入库管理**：物料的出库、入库、租用
- **审批流程**：物料申请的审批

### 3.4 薪资管理模块
- **薪资计算**：基于工作记录自动计算薪资
- **薪资发放**：支持单条和批量发放
- **薪资查询**：个人薪资记录查询
- **统计报表**：薪资支出统计

### 3.5 财务管理模块
- **收支统计**：活动收入、物料支出、人员成本
- **利润分析**：活动利润计算
- **报销管理**：费用报销申请和审批
- **财务核验**：收入确认和核验

### 3.6 考勤管理模块
- **工作记录**：记录员工工作情况
- **考勤统计**：月度考勤统计
- **状态跟踪**：工作完成状态跟踪

## 4. 业务流程

### 4.1 活动申请流程
```
用户申请活动
    ↓
业务管理员审批
    ↓ (通过)
仓库管理员审批物料
    ↓ (通过)
安排工作人员
    ↓
用户支付定金
    ↓
活动执行
    ↓
用户支付尾款
    ↓
活动完成
```

### 4.2 薪资发放流程
```
系统计算薪资
    ↓
生成薪资记录
    ↓
财务人员审核
    ↓
薪资发放
    ↓
更新发放状态
    ↓
员工查看薪资
```

## 6. 系统特点

### 6.1 技术特点
- **前后端分离**：前端使用jQuery AJAX与后端交互
- **RESTful风格**：接口设计符合RESTful规范
- **数据安全**：使用POST请求传输敏感数据
- **编码统一**：UTF-8编码支持中文

### 6.2 业务特点
- **流程完整**：覆盖活动从申请到完成的全流程
- **多级审批**：支持多级审批机制
- **权限精细**：基于角色的细粒度权限控制
- **实时跟踪**：活动状态实时更新和通知

### 6.3 用户体验
- **响应式设计**：适配不同屏幕尺寸
- **友好提示**：清晰的错误提示和操作引导
- **数据可视化**：图表展示统计数据
- **操作便捷**：简洁直观的操作界面

## 7. 部署说明

### 7.1 环境要求
- **操作系统**：Windows 10/11 或 Linux
- **JDK版本**：JDK 17 或更高
- **数据库**：MySQL 8.0
- **Web服务器**：Apache Tomcat 9.x
- **浏览器**：Chrome 90+、Firefox 88+、Edge 90+

### 7.2 部署步骤
1. 安装JDK并配置环境变量
2. 安装MySQL并创建数据库
3. 导入SQL脚本创建数据表
4. 安装Tomcat服务器
5. 将项目部署到Tomcat
6. 配置数据库连接参数
7. 启动Tomcat服务
8. 访问系统首页

### 7.3 访问地址
- 首页：`http://localhost:8080/EventPlanning_Management/html/index.html`
- 登录页：`http://localhost:8080/EventPlanning_Management/html/index.html`
