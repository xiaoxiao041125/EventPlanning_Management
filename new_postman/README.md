# 活动策划管理平台接口测试文档

## 目录结构

```
new_postman/
├── event_planning_collection.json  # Postman 测试集合文件
├── login_test_data.json            # 登录接口参数化测试数据
├── apply_activity_test_data.json   # 申请活动接口参数化测试数据
├── register_test_data.json         # 注册接口参数化测试数据
├── run_tests.bat                   # 运行测试的批处理脚本
└── README.md                       # 使用说明文档
```

## 测试覆盖范围

本测试集合覆盖了以下模块的所有接口：

1. **用户管理**
   - 登录接口
   - 注册接口
   - 修改密码接口
   - 更新用户信息接口
   - 获取所有用户接口
   - 根据ID获取用户接口

2. **活动管理**
   - 申请活动接口
   - 取消活动接口
   - 更新活动进度接口
   - 获取所有活动需求接口
   - 根据ID获取活动需求接口
   - 根据用户ID获取活动需求接口

3. **员工管理**
   - 员工注册接口
   - 处理员工注册接口
   - 获取所有员工接口

4. **薪资管理**
   - 计算薪资接口
   - 发放薪资接口
   - 获取我的薪资接口
   - 获取所有薪资记录接口

5. **物料管理**
   - 添加物料接口
   - 物料出库接口
   - 物料租赁接口
   - 获取仓库物料接口

6. **报销管理**
   - 添加报销申请接口
   - 审批报销申请接口
   - 获取所有报销申请接口
   - 根据用户ID获取报销申请接口

## 参数化测试

对于包含参数的接口，已创建参数化测试数据，覆盖：
- 正常值

## 使用方法

### 1. 安装依赖

需要安装以下工具：

```bash
# 安装 newman 和 allure 报告插件
npm install -g newman newman-reporter-allure

# 安装 Allure 命令行工具
# 从 https://github.com/allure-framework/allure2/releases 下载并添加到环境变量
```

### 2. 运行测试

1. 确保系统已经启动
2. 双击运行 `run_tests.bat` 脚本
3. 脚本会自动运行所有测试并生成 Allure 报告
4. 测试完成后会自动打开 Allure 报告

### 3. 查看测试报告

Allure 报告包含：
- 测试执行结果
- 测试覆盖率
- 详细的测试步骤
- 响应时间统计
- 失败测试的详细信息

## 测试数据说明

### 登录接口测试数据
- 正常登录：管理员账号、普通用户
- 异常登录：空用户名、空密码、空角色、错误密码、错误角色、不存在的用户

### 申请活动接口测试数据
- 正常申请：完整参数
- 异常申请：空用户ID、空活动地点、空活动类型、开始时间晚于结束时间、参与人数为0、空需求描述
- 边界值测试：参与人数过多、预算过高

### 注册接口测试数据
- 正常注册：普通用户、管理员
- 异常注册：空用户名、空密码、空角色、用户名已存在
- 边界值测试：用户名过短、用户名过长、密码过短

## 注意事项

1. 测试前确保系统已经启动
2. 确保数据库连接正常
3. 测试过程中可能会产生测试数据，请在测试完成后清理
4. 对于需要权限的接口，请确保使用正确的角色进行测试
