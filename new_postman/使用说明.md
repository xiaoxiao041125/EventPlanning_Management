# Newman 参数化测试执行说明

## 🚀 快速开始（推荐方式）

### 方式一：使用 PowerShell 脚本（最推荐）

在 `new_postman` 目录下双击运行：
```
run_all_tests.ps1
```

或者在 PowerShell 命令行中：
```powershell
cd e:\毕业论文\Java_Project\EventPlanning_Management\new_postman
.\run_all_tests.ps1
```

**特点**：
- ✅ 为每个接口自动使用正确的参数化数据文件
- ✅ 彩色输出，清晰显示每个接口的测试结果
- ✅ 显示测试总结（通过/失败数量）
- ✅ 测试结束后不会立即关闭窗口

---

### 方式二：使用批处理脚本（生成 HTML 报告）

```bash
cd e:\毕业论文\Java_Project\EventPlanning_Management\new_postman
run_tests.bat
```

**特点**：
- ✅ 自动打开 HTML 报告
- ✅ 适合需要查看详细报告的场景
- ⚠️ 注意：此方式不使用参数化数据文件，可能部分测试会失败

---

### 方式三：直接使用 Newman 命令

```bash
# 运行整个集合（不使用参数化数据）
newman run event_planning_collection.json --delay-request 100

# 运行单个接口（带参数化数据）
newman run event_planning_collection.json --folder "用户管理/登录接口" -d login_test_data.json --delay-request 100
```

## ❓ 常见问题

### Q1: 为什么在 Postman 上测试通过，但跑脚本失败？

**原因**：Postman 手动测试时使用的是单个数据，而 Newman 运行脚本时需要通过参数化数据文件提供测试数据。

**解决方案**：使用 `run_all_tests.ps1` 脚本，它会自动为每个接口匹配正确的数据文件。

---

### Q2: 为什么显示 "Unable to find a folder" 错误？

**原因**：`--folder` 参数中的中文文件夹名称编码问题。

**解决方案**：
1. 使用 PowerShell 运行 `run_all_tests.ps1`
2. 或者在命令行中使用英文引号包裹文件夹名称

---

### Q3: 为什么 CMD 显示乱码？

**原因**：CMD 默认使用 GBK 编码，而脚本是 UTF-8 编码。

**解决方案**：使用 PowerShell 运行脚本，PowerShell 对 UTF-8 编码支持更好。

---

### Q4: 如何生成 HTML 报告？

使用以下命令：
```bash
newman run event_planning_collection.json --delay-request 100 --reporters cli,html --reporter-html-export report.html
```

或者直接运行 `run_tests.bat`，会自动生成并打开 HTML 报告。

---

### Q5: Newman 未找到？

运行以下命令安装：
```bash
npm install -g newman
```

验证安装：
```bash
newman --version
```

## 📝 输出说明

测试执行后会显示：
- ✅ **通过的测试**（绿色）：接口返回符合预期
- ❌ **失败的测试**（红色）：接口返回不符合预期或请求失败

## ⚠️ 注意事项

1. **确保 Tomcat 服务器运行**：测试前请确保 Tomcat 已启动且项目已部署
2. **确保数据库连接正常**：测试数据依赖数据库中的数据
3. **确保 Newman 已安装**：建议使用 v6.x 版本
4. **确保测试数据文件存在**：所有 *_test_data.json 文件应在同一目录下
