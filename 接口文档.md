## 一、通用说明

### 1.1 接口地址格式

```
http://服务器地址:端口/EventPlanning_Management/接口路径
```

### 1.2 通用响应格式

所有接口统一返回以下JSON格式：

```json
{
    "data": "返回数据或提示信息",
    "flag": "success/fail",
    "message": "可选的附加信息"
}
```

### 1.3 响应状态说明

| flag值 | 说明 |
|--------|------|
| success | 操作成功 |
| fail | 操作失败 |

### 1.4 通用错误码

| 错误信息 | 说明 |
|----------|------|
| 请填写用户名、密码和角色 | 登录/注册参数缺失 |
| 错误: 缺少必填参数 | 必要参数未提供 |
| 错误: 参数格式不正确 | 参数类型错误 |
| 错误: 记录不存在 | 查询的记录不存在 |

### 3.5 更新活动进度

- **接口地址**: `/UpdateActivityProgress`
- **请求方式**: POST
- **功能说明**: 更新活动审批状态

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| demandId | String | 是 | 活动需求ID |
| activityProgress | String | 是 | 进度状态 (-2到5) |
| refuseReason | String | 否 | 拒绝原因 |
| principalUserId | String | 否 | 负责人ID |

**活动进度状态说明**:

| 状态值 | 说明 |
|--------|------|
| -3 | 用户已取消 |
| -2 | 仓库管理员拒绝 |
| -1 | 业务管理员拒绝 |
| 0 | 待处理 |
| 1 | 业务管理员通过 |
| 2 | 仓库管理员通过 |
| 3 | 人员安排完成 |
| 4 | 用户已支付定金 |
| 5 | 用户已支付尾款/已完成 |

// 使用HttpURLConnection发送POST请求
URL url = new URL("http://localhost:8080/EventPlanning_Management/login");
HttpURLConnection conn = (HttpURLConnection) url.openConnection();
conn.setRequestMethod("POST");
conn.setDoOutput(true);
conn.setRequestProperty("Content-Type", "application/x-www-form-urlencoded");

String params = "username=zhangsan&password=123456&roleId=1";
try (OutputStream os = conn.getOutputStream()) {
    os.write(params.getBytes(StandardCharsets.UTF_8));
}

// 读取响应
BufferedReader reader = new BufferedReader(new InputStreamReader(conn.getInputStream()));
String line;
StringBuilder response = new StringBuilder();
while ((line = reader.readLine()) != null) {
    response.append(line);
}
System.out.println(response.toString());
